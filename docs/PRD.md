# 노션 기반 견적서(Invoice) 웹 애플리케이션 MVP PRD

## 🎯 핵심 정보

**목적**: Notion 데이터베이스에 입력한 견적서를 클라이언트가 웹에서 확인하고 PDF로 다운로드받을 수 있도록 제공
**사용자**: 영업팀(견적서 작성/Notion 입력), 클라이언트(웹에서 조회/PDF 다운로드)

---

## 🚶 사용자 여정

### 영업팀 여정

```
1. 로그인 페이지
   ↓ [이메일/비밀번호 입력]

2. 견적서 목록 페이지
   ↓ [검색 및 필터링]

3. 견적서 상세 페이지 (확인 용도)
   ↓ [뒤로가기 또는 다른 견적서 조회]
```

### 클라이언트 여전

```
1. 로그인 페이지
   ↓ [이메일/비밀번호 입력]

2. 견적서 목록 페이지
   ↓ [자신의 견적서 목록 조회 (고객명 필터링)]

3. 견적서 상세 페이지
   ↓ [PDF 다운로드 클릭]

4. PDF 다운로드 완료
   ↓ [목록으로 돌아가기]
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명                | 설명                                                             | MVP 필수 이유                  | 관련 페이지        |
| -------- | --------------------- | ---------------------------------------------------------------- | ------------------------------ | ------------------ |
| **F001** | Notion 데이터 연동    | Notion 데이터베이스에서 견적서 데이터를 실시간으로 조회          | 견적서 데이터의 단일 소스 제공 | 견적서 목록 페이지 |
| **F002** | 견적서 목록 조회      | 모든 견적서를 목록으로 표시 (제목, 고객명, 작성일, 상태 등)      | 사용자의 기본 필요 기능        | 견적서 목록 페이지 |
| **F003** | 견적서 검색 및 필터링 | 견적서 번호, 고객명, 상태, 날짜 범위 등으로 검색/필터링          | 수많은 견적서 중 빠른 조회     | 견적서 목록 페이지 |
| **F004** | 견적서 상세 정보 조회 | 견적서의 전체 정보 표시 (고객정보, 상품 목록, 금액, 유효기간 등) | 견적서 검토 및 확인            | 견적서 상세 페이지 |
| **F005** | PDF 다운로드          | 견적서를 PDF 형식으로 다운로드                                   | 클라이언트의 핵심 요구사항     | 견적서 상세 페이지 |

### 2. MVP 필수 지원 기능

| ID       | 기능명                     | 설명                                                         | MVP 필수 이유                     | 관련 페이지                            |
| -------- | -------------------------- | ------------------------------------------------------------ | --------------------------------- | -------------------------------------- |
| **F010** | 기본 인증                  | 회원가입/로그인/로그아웃 기능                                | 서비스 접근 및 데이터 보안        | 로그인 페이지, 회원가입 페이지         |
| **F011** | 사용자 역할 기반 접근 제어 | 영업팀은 모든 견적서 조회, 클라이언트는 자신의 견적서만 조회 | 데이터 보안 및 개인정보 보호      | 견적서 목록 페이지, 견적서 상세 페이지 |
| **F012** | 견적서 유효성 표시         | 유효기간 기준으로 견적서 상태 시각화 (유효/만료)             | 사용자의 빠른 판단 및 업무 효율성 | 견적서 목록 페이지, 견적서 상세 페이지 |

### 로그인 페이지

> **구현 기능:** `F010` | **메뉴 위치:** 헤더 > 로그인

| 항목            | 내용                                                                                                                                                 |
| --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 사용자 인증 전용 페이지                                                                                                                              |
| **진입 경로**   | 비로그인 상태에서 로그인 메뉴 클릭, 또는 자동 리디렉션                                                                                               |
| **사용자 행동** | 이메일과 비밀번호를 입력하고 로그인 버튼 클릭                                                                                                        |
| **주요 기능**   | • 이메일 입력 필드<br>• 비밀번호 입력 필드<br>• 이메일 형식 유효성 검사<br>• 로그인 실패 시 에러 메시지 표시<br>• **로그인** 버튼<br>• 회원가입 링크 |
| **다음 이동**   | 로그인 성공 → 견적서 목록 페이지, 로그인 실패 → 에러 메시지 표시 후 현재 페이지 유지                                                                 |

| 항목            | 내용                                                                                                                                                                                                                                                                            |

| **사용자 행동** | 이메일, 비밀번호, 이름, 역할(영업팀/클라이언트)을 입력하고 회원가입 버튼 클릭                                                                                                                                                                                                   |
| **주요 기능**   | • 이메일 입력 필드<br>• 비밀번호 입력 필드<br>• 비밀번호 확인 필드<br>• 사용자 이름 입력 필드<br>• 역할 선택 드롭다운 (영업팀/클라이언트)<br>• 고객명 입력 필드 (클라이언트 선택 시)<br>• 입력 유효성 검사<br>• 중복 이메일 검사<br>• **회원가입** 버튼<br>• 로그인 페이지 링크 |
| **다음 이동**   | 회원가입 성공 → 로그인 페이지 (자동 리디렉션), 회원가입 실패 → 에러 메시지 표시                                                                                                                                                                                                 |

### 견적서 목록 페이지

> **구현 기능:** `F001`, `F002`, `F003`, `F011`, `F012` | **메뉴 위치:** 헤더 > 홈

| **사용자 행동** | 견적서 목록을 스크롤 조회, 검색 및 필터링으로 특정 견적서 찾기, 견적서 클릭하여 상세 페이지 이동                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **주요 기능**   | • Notion 데이터베이스에서 견적서 데이터 실시간 로드<br>• 견적서 목록 테이블 표시 (견적서 번호, 고객명, 금액, 상태, 작성일, 유효기간)<br>• **검색 기능**: 견적서 번호 또는 고객명으로 실시간 검색<br>• **필터링**: 상태(Draft/Sent/Accepted/Expired), 날짜 범위, 통화(KRW/USD)<br>• **역할별 필터링**: 영업팀은 모든 견적서, 클라이언트는 자신의 고객명 견적서만 조회<br>• **상태 시각화**: 유효기간 기준으로 상태 아이콘 표시 (유효=녹색, 만료=회색)<br>• 각 행 클릭 → 상세 페이지 이동<br>• 페이지네이션 또는 무한 스크롤 |
| **다음 이동**   | 견적서 클릭 → 견적서 상세 페이지, 헤더 로그아웃 클릭 → 로그인 페이지                                                                                                                                                                                                                                                                                                                                                                                                                                                       |

### 견적서 상세 페이지

> **구현 기능:** `F001`, `F004`, `F005`, `F011`, `F012` | **메뉴 위치:** 견적서 목록에서 항목 클릭

| 항목            | 내용                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 선택한 견적서의 모든 상세 정보를 표시하고 PDF 다운로드 제공                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| **진입 경로**   | 견적서 목록 페이지에서 특정 견적서 행 클릭                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **사용자 행동** | 견적서의 전체 정보를 확인하고, 필요시 PDF로 다운로드 버튼 클릭                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **주요 기능**   | • Notion에서 해당 견적서의 모든 데이터 로드<br>• **고객 정보 섹션**: 고객명, 고객 이메일, 연락처(표시됨)<br>• **견적서 기본 정보 섹션**: 견적서 번호, 작성일, 유효기간, 상태<br>• **상품 목록 섹션**: 상품명, 수량, 단가, 금액을 테이블로 표시<br>• **금액 요약 섹션**: 소계, 세금(표시됨), 총 금액, 통화(KRW/USD)<br>• **상태 표시**: 유효기간 기준으로 상태 시각화<br>• **비고 섹션**: Notion의 비고 내용 표시<br>• **PDF 다운로드** 버튼 (클라이언트만 활성화, 영업팀도 가능하지만 주요 기능은 클라이언트)<br>• **뒤로가기** 버튼 → 견적서 목록으로 이동 |
| **다음 이동**   | 뒤로가기 버튼 클릭 → 견적서 목록 페이지, PDF 다운로드 → PDF 파일 다운로드 시작                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |

## 🗄️ 데이터 모델

### User (사용자)

| 필드          | 설명                            | 타입/관계           |
| ------------- | ------------------------------- | ------------------- |
| id            | 고유 식별자                     | UUID                |
| email         | 이메일                          | String (unique)     |
| password      | 암호화된 비밀번호               | String              |
| name          | 사용자 이름                     | String              |
| role          | 사용자 역할 (영업팀/클라이언트) | Enum: SALES, CLIENT |
| customer_name | 고객명 (클라이언트인 경우만)    | String (nullable)   |
| created_at    | 가입일                          | Timestamp           |
| updated_at    | 수정일                          | Timestamp           |

### Invoice (견적서 - Notion 동기화)

| 필드           | 설명                               | 타입/관계                            |
| -------------- | ---------------------------------- | ------------------------------------ |
| id             | Notion 데이터베이스 ID             | String                               |
| invoice_number | 견적서 번호                        | String (unique)                      |
| customer_name  | 고객명                             | String                               |
| customer_email | 고객 이메일                        | String                               |
| total_amount   | 총 금액                            | Decimal                              |
| currency       | 통화 (KRW/USD)                     | Enum: KRW, USD                       |
| status         | 상태 (Draft/Sent/Accepted/Expired) | Enum: DRAFT, SENT, ACCEPTED, EXPIRED |
| created_date   | 작성일                             | Date                                 |
| expiry_date    | 유효기간                           | Date                                 |
| notes          | 비고                               | Text (nullable)                      |
| synced_at      | 마지막 동기화 시간                 | Timestamp                            |

### InvoiceItem (견적서 상품)

| 필드       | 설명               | 타입/관계    |
| ---------- | ------------------ | ------------ |
| id         | 고유 식별자        | UUID         |
| invoice_id | 견적서 ID          | → Invoice.id |
| item_name  | 상품명             | String       |
| quantity   | 수량               | Decimal      |
| unit_price | 단가               | Decimal      |
| amount     | 금액 (수량 × 단가) | Decimal      |

## 🛠️ 기술 스택 (최신 버전)

### 🎨 프론트엔드 프레임워크
